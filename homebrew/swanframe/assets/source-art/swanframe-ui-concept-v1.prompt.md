# SWANFRAME player UI concept prompt — v1

Mode: OpenAI ImageGen built-in tool, reference-guided UI generation.

References: the working SWANFRAME ROM screenshot and the approved original v5
SWANFRAME mascot source art. The screenshot supplied information architecture;
the mascot supplied character identity and palette.

## Prompt

Create a practical 224×144 WonderSwan Color music-player interface reference.
Replace the text-heavy terminal layout with a compact cockpit music deck:

- A cropped SWANFRAME mascot portrait occupies roughly the left third.
- One dominant track card contains a large track number, short title, segmented
  progress timeline, BPM/speed/volume chips, and strong negative space.
- Four bold horizontal channel lanes use waveforms, pulse shapes, meters, and
  icons instead of instrument prose.
- A small icon-led button dock replaces instructional sentences.
- Use crisp, tile-aware late-1990s Japanese handheld sci-fi styling: flat amber
  and cyan over near-black navy, high contrast, thick one-pixel dividers, and
  four-color-friendly regions.

Conceptual text was restricted to SWANFRAME, 01/05, BATTERY GLOW, 148, and 100%,
plus tiny A/B/X/Y and 1–4 button markers. The native ROM renders all final text
and reconstructs the hierarchy with tiles; this generated image is an art and
layout reference, not a pre-rendered game screen.

The mascot must remain the original amber/gunmetal/cyan SWANFRAME design. No
Gundam or other franchise names, logos, V-fin, watermark, desktop window chrome,
photorealism, dense prose, fake keyboard, or extra characters.
